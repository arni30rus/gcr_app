GCR APP (Gym Client Registration)

Кроссплатформенное приложение для учета клиентов спортзала, разработанное на Flutter. Работает на Windows (10/11) и Android (10+).

Приложение спроектировано с приоритетом офлайн-работы (Offline-first): все данные всегда доступны локально, а синхронизация с облаком происходит по таймеру или по кнопке.


🚀 Основные возможности

- Сканер абонементов: Поддержка USB/HID сканеров на Windows и камеры смартфона на Android + ручной ввод.
- Гибкая логика абонементов: Настраиваемые типы (время, дни недели, VIP-статус). Статус рассчитывается на лету.
- Мультиарендность (Залы): Одно облако Supabase может обслуживать несколько независимых залов. Данные изолированы.
- Локальный режим (Офлайн): Полноценная работа без интернета с возможностью экспорта/импорта базы данных (.db).
- Дашборд: Статистика посещений, активных абонементов и скоро истекающих карт.
- Soft Delete: Удаленные клиенты скрываются из списка, но остаются в базе для возможного восстановления.
-  Экспорт в CSV: Выгрузка списка активных клиентов для Excel/рассылок.

🛠 Технологии и сборка

- Фреймворк: Flutter (Dart)
- Локальная БД: SQLite (sqflite / sqflite_common_ffi)
- Облачная БД: Supabase (PostgreSQL)
- Авторизация: Supabase Auth + Row Level Security (RLS)
 
1. Для сборки установщика для Android запустите команду:
```bash
flutter build apk
```
APK файл будет лежать по следующему адресу ~\gym_app\build\app\outputs\apk\release

2. Для сборки версии для Windows:
```bash
flutter build windows
```
Папка с библиотеками и файлом для запуска будет лежать по следующему адресу: ~\gym_app\build\windows\x64\runner\Release
Если хотите сделать установщик exe, то установите ПО Inner Setup Compiler и запустите файл installer.iss в папке проекта.
Готовый exe файл будет лежать в папке проекта в подпапке InstallerOutput.

📱 Как пользоваться (Для владельца зала)

1. При первом запуске выберите режим работы:
    - Основное облако — если у вас есть учетка (админ проекта выдает доступ другим залам).
    - Локальная база — работа полностью без интернета, бэкапы делаются руками через Экспорт/Импорт.
    - Своя база — если вы сами зарегистрировали Supabase по инструкции.
2. Во вкладке "Абонементы" создайте типы карт (Дневной, Вечерний и т.д.), указав время и дни работы.
3. Во вкладке "Сканер" сканируйте карты клиентов при входе. Если карты нет в базе, приложение предложит создать клиента.

📄 Лицензия

Данный проект распространяется как есть. Используйте открытый код на свой страх и риск. 
Все используемые библиотеки (Flutter, Supabase, SQLite) являются open-source.

P.S. На моем операторе интернета без VPN SUPABASE не работает. Используйте "обходные пути". Можно вести локальную базу, а после того как у вас будет стабильно работать SUPABASE сделать экспорт локальной базы, подключиться к своей базе в SUPABASE и загрузить туда локальную БД.
